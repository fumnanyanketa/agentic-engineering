🎯 Objective: Learn to keep an agent coherent across long tasks that take many steps and span more text than fits in a single context window. By the end you will have built the two core techniques yourself: compaction and an external memory file.

Recall from Module 1 that the context window is the model's short-term working memory: the largest amount of text, counted in tokens (the small chunks of text models read), that it can look at in one request. For a quick chat that is plenty. But an agent working on a real task, calling tools, reading files, and trying things over hundreds of steps, will produce far more text than any window holds. When the window fills up, older content must be dropped, and a naive agent simply forgets what it was doing. This module is about not forgetting.

Two kinds of memory

It helps to separate two ideas that beginners often blur together.

  • Short-term memory is the live context window: the current conversation and the working set of facts the agent is using right now. It is fast but small and temporary.
  • Long-term memory is information saved outside the window in durable storage (a file or a database) so it survives even after the window is cleared. Good systems often write to long-term memory in the background, so saving a memory does not stall the main loop. (Rethinking memory mechanisms of foundation agents, 2026 survey)

A crucial point: a bigger context window is not the same as long-term memory. A larger window is still short-term, still finite, and still gets wiped between sessions. Long-horizon coherence comes from how you manage memory, not from window size alone. (Less context, better agents, 2026)

Compaction: summarize and restart

The first core technique is compaction. As the conversation approaches the window limit, you pause, ask the model to summarize what has happened so far, and then start a fresh window seeded with that summary instead of the full history. A good summary keeps the things that matter (decisions made, the current goal, open problems, key facts learned) and throws away the noise (long raw tool outputs that have already been used, repeated boilerplate).

Think of it like clearing a cluttered desk: you write the important notes on one clean page, sweep the rest into the bin, and keep working. Done well, the agent continues smoothly across the reset. The danger, covered in the pitfalls below, is summarizing away the one detail you turn out to need later, so be deliberate about what the summary must always preserve. (Effective context engineering for AI agents)

Structured note-taking: an external scratchpad

The second core technique is to let the agent keep notes in a file outside the window, a kind of scratchpad it can read and write at any time. Instead of trying to hold everything in its head, the agent writes down progress, intermediate results, and a running checklist, then reloads those notes after a compaction or in a later session. This is exactly how some long-running agent demos have stayed on track across thousands of steps and many context resets: they kept tallies and plans in durable external notes rather than in the window. (Effective context engineering for AI agents)

Sub-agent isolation

A third technique, which previews the multi-agent work in Phase 4, is to hand a big self-contained sub-task to a separate sub-agent that runs in its own clean context window. The sub-agent does all the messy work in isolation and returns only a short, distilled summary (often just one or two thousand tokens) to the main agent. The main window stays uncluttered, because all the intermediate mess lived and died in the sub-agent's window. Compressing past observations and traces into compact summaries this way is one of the most consistent findings in recent long-horizon agent research. (Less context, better agents, 2026)

Treat context as a budget

The unifying mindset: your context window is a budget, and every token spent is a token unavailable for actual reasoning. Keep a running ledger in your head of what is in the window and why. Persist the things worth keeping to long-term memory, drop the noise, and isolate large sub-tasks. Research that makes this budget explicit finds that agents which neither over-compress (erasing critical evidence) nor under-compress (overflowing the limit) perform best, which is just a formal way of saying: spend the budget wisely. (ContextBudget, budget-aware context management, 2026)

One practical note on tooling. Many agent frameworks now offer context management, session handling, and sub-agents built in, so you will not always write this by hand in production. Tools you might encounter include framework features for sessions and memory from various vendors. Even so, build a manual compaction loop and a manual scratchpad once yourself first. Understanding what the framework is doing under the hood is what lets you debug it when it misbehaves.

Lab 6: Extend your assistant so it can survive a task that is too long for its context window. Implement compaction (summarize the conversation, then restart the window from the summary) and a simple external scratchpad file the agent reads from and writes to. Then give it a task that genuinely cannot fit in one window, and show it completing the task across at least one compaction by relying on its scratchpad.

✅ Best practices:

  • Treat the context window as a budget and keep a mental ledger of what is in it.
  • Persist decisions and key facts to durable memory, and drop redundant noise.
  • Isolate large sub-tasks into their own windows so the main window stays clean.
  • Be explicit about what a summary must always preserve before you compact.

⛔ Pitfalls:

  • Letting the message history grow forever until the window overflows.
  • Summarizing away the one detail that turns out to matter.
  • Confusing a long context window with true long-horizon memory.
  • Over-compressing and erasing critical evidence, or under-compressing and overflowing.

🏁 Milestone: You have an agent that completes a task spanning multiple context windows by combining compaction with an external memory file.
