🎯 Objective: Compose everything into a single, real, evaluated, observable, hardened agent-orchestration system — and document the build in public.

This is the diploma. There is no new concept here; there is integration, which is its own skill. Build it using the coding-agent craft from Module 16 — that's the point of learning it first.

Required architecture (the exit bar): 1. Orchestration — a lead/orchestrator agent that decomposes a non-trivial task and delegates to specialized workers (Modules 7, 10), with a reasoned, cost-aware choice of single- vs. multi-agent (Module 10). 2. Tools via MCP — capabilities exposed through MCP, including at least one server you wrote (Modules 4, 9). 3. Retrieval + memory — hybrid retrieval as a tool, plus compaction/external memory for long horizons (Modules 5, 6). 4. The agent loop done right — bounded iterations, verification/self-check steps, sane stopping conditions (Module 8). 5. An eval suite — built from real failures, with a reported pass rate per component (Module 12). 6. Observability — full tracing, per-step cost/latency (Module 13). 7. Safety — least-privilege, human-gated consequential actions, and a documented defense against the lethal trifecta (Module 14). 8. Hardening — per-task cost/latency budget, tier-optimized, cached, with fallbacks (Module 15).

Suggested capstone domains (pick one you'll actually use — dogfooding is the point): - A research orchestrator (breadth-first, the multi-agent sweet spot) that answers deep questions with cited synthesis. (Note: this curriculum was itself built by exactly such a system — that's a publishable meta-story.) - A content/publishing pipeline for AI Nativity (route → research → draft → evaluate-optimize → schedule) — directly serves the build-the-habit goal. - A personal-knowledge agent over your notes/archive (ties to the Anchor tracker idea).

Capstone deliverables: 1. The working system (repo). 2. A system-design doc explaining each architectural choice and its trade-off (esp. single- vs. multi-agent, framework-or-not). 3. An eval report with pass rates. 4. A cost/latency report per task. 5. A security note documenting the trifecta audit and mitigation. 6. A build-in-public series (one post per phase — you've been drafting these all along via "Publish this").

🏁 Final milestone — you are an agent orchestrator when you can: stand up a system that decomposes work across agents, ground it in tools/data via MCP, manage context across long horizons, prove its reliability with evals and traces, defend it against injection, and state its cost per task — and explain every design decision and its alternative. That's not a hobby project. That's the job.

✎ Publish this: the whole capstone, as a series. "I taught myself agentic engineering in 16 weeks and built an agent-orchestration system. Here's everything." (The series is the proof of the skill — and the AI Nativity flagship.)
