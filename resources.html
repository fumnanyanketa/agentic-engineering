Appendix B — Quick-reference decision rules

  • Workflow or agent? Predictable steps → workflow. Unpredictable steps/horizon → agent. Default to the simplest that works.
  • Single or multi-agent? Read-heavy/breadth-first/parallelizable/high-value → multi-agent. Write-heavy/dependent/sequential → single agent. Always compare cost, not just quality (multi-agent ≈ 15× tokens).
  • Prompt, RAG, or fine-tune? Prompt first → add RAG when it needs your/fresh data → fine-tune only for style/voice/narrow-high-volume or when prompt+RAG miss the bar.
  • Which model tier? Cheapest tier that passes the eval. Re-check the eval after any cost optimization.
  • Framework or raw API? Learn raw first. Adopt a framework for convenience once you understand the loop, chosen by problem shape.
  • Vibe coding or agentic engineering? If you're shipping agent-written code you haven't reviewed, that's vibe coding. Agentic engineering = a verification loop (tests/types) + reviewing every diff + sandboxed execution. You stay accountable for the output.
  • Is it safe? Audit for the lethal trifecta. If all three legs are present, redesign to break one. Human-gate consequential actions. Sandbox any agent that executes code.
  • Is it done? Not until it has an eval with a reported pass rate. "It works" without an eval is a vibe, not an engineering claim.

Appendix C — Honest caveats about this curriculum

  • The field moves fast. Tooling specifics (frameworks, vector DBs, model IDs/pricing, even MCP's governance) will drift. The concepts (Phases 1–5 vendor-neutral layers) are durable; re-verify the Claude-layer and tooling specifics against current docs before relying on them.
  • Some sources are vendor marketing. Eval-platform and framework rankings come from competing companies' blogs. They're directionally useful, not neutral benchmarks. Primary sources (Appendix A) carry the real weight.
  • Genuine disagreements are left intact, on purpose. Multi-agent (Anthropic vs. Cognition) and evals (EDD vs. error-analysis-first) are unsettled. A curriculum that pretended otherwise would be lying to you. Form your own view by building.
  • Numbers are illustrative unless from a primary source. The p^N math, the "32k-token context rot" figure, the "+90.2%" multi-agent result, token-cost multipliers — treat as directional signals, and cite their context if you republish.
