✦The destination

If you follow this course and build the projects, you go from "I use Claude to ask questions" to "I can take an idea and ship a production grade, self improving AI agent, deploy it on real infrastructure, and lead a team doing the same." You will be fluent in the Claude ecosystem and, more importantly, in the durable disciplines that outlast any single tool: prompting, evaluation, context engineering, agent design, and verification.

You also finish with proof. There are 37 hands on projects. Build them and you end with roughly 37 working artifacts in a public repository. Knowledge fades. A portfolio of things you built does not.

Your north-star project

AtlasOS: a self-improving operating system of cooperating AI agents

You build AtlasOS, the platform you would give an AI-native company: a fleet of agents (the Atlas orchestrator dispatching Scout, Forge, Pulse, Herald, and Warden over a shared Cortex memory) that you hand an outcome and it plans, executes, verifies itself, remembers, and improves. Bend every lesson's capstone toward it. By the end you will not have 38 disconnected exercises. You will have shipped one real platform. The full brief, architecture map, and lesson-to-component roadmap live in the atlas/ folder of the repo.

It is the perfect target because it touches every module:

  • Prompting and models: the agents' prompts and the model-routing (advisor) policy.
  • Evals: Warden's harness that grades whether each agent's output is actually good.
  • Claude Code: build the orchestrator and dev loop that runs the fleet.
  • Managed Agents: ship your first production agent (Scout) with memory.
  • Memory and dreaming: Cortex learns and improves the fleet over time.
  • Cloud and deployment: run AtlasOS on real infrastructure so it survives a closed laptop.
  • Leadership and case studies: the operating model, plus one flagship vertical agent, in public.

Not feeling it? Two alternates that work just as well: a research-org agent fleet, or a support-org agent stack. Pick one ambitious, module-spanning build and commit to it.

✦The daily ritual (about two hours)

One lesson a day is very doable in two focused hours. Here is the loop that makes each day count:

  1. Watch the talk (15 to 35 min). Each lesson links the original video at the top.
  2. Read the lesson (20 to 30 min). Check off the learning objectives as you go.
  3. Build the capstone (45 to 75 min), and build it as the AtlasOS component for that lesson, not the toy example.
  4. Log it in public (5 to 10 min). Commit your work and write two or three sentences on what you learned and what broke.
  5. Tick the lesson off in your tracker. Momentum is the whole game.

At 37 lessons, this is about 6 to 8 weeks with rest days. Treat weekends as catch up or rest, not guilt.

✦How to track it

You are doing everything inside Claude Code, so your tracker lives there too. In the repo there is a PROGRESS.md file: a checklist of all 37 lessons plus a short daily log. Each day, ask Claude Code to tick the lesson and add your log line, then commit. The lesson pages themselves also remember which objectives and milestones you have checked off in your browser.

Best case

In 6 to 8 weeks you have shipped AtlasOS, you have a fleet of working agents and a daily build log, and you can credibly call yourself a capable AI engineer and agent builder. You did not just learn it. You proved it, in public.

Worst case

You watch passively and skip the builds, capstones balloon until you stall, a missed day becomes a missed week, and setup or cost friction kills momentum. You end with notes and no portfolio.

✦The rules that decide which one you get

  • Protect the time slot. Same two hours every day beats heroic weekend marathons.
  • Timebox the capstone. If it grows past your slot, ship a smaller version and move on. A shipped small thing beats a perfect unfinished thing.
  • Always build, never just read. Have your API key and Claude Code ready before Lesson 1. New to the tools, or rusty? Do the optional Module 0 pre-flight first, it gets your whole toolchain working in one sitting.
  • One missed day is fine. Two in a row is the real risk. If you slip, do a tiny ten minute lesson rather than nothing, to keep the streak alive.
  • Learn the principles, hold the tools loosely. Model names and features will change. Prompting, evals, context engineering, agent design, and verification will not.
The course is built to be taken in order. It runs Foundations, then core model skills, then evals, then Claude Code, then building agents, then advanced agent engineering, then deployment, then leadership, then real world case studies. Each stage assumes the one before it. If you are new to the tools or rusty, start with the optional Module 0 pre-flight; otherwise start at Lesson 1 and go straight through.
